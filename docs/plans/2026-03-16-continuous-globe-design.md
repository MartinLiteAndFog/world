# Continuous Globe Interaction Design

**Status:** Approved

**Goal:** Define a single-scene globe interaction model that starts with global thematic overlays and continuously transitions into country analysis, industry focus, clustered markers, and raw business points.

## Decision Summary

- Chosen direction: single continuous globe state
- First prototype data strategy: real country boundaries with mocked business density/value payloads
- Country selection behavior: fade the rest of the world and zoom toward the selected country
- Country analysis default: an `activity map` generated by colored points rather than a separate heatmap raster
- Activity map encoding: density shifts hue toward red; value increases brightness
- Industry behavior: keep the all-industry activity map as the base layer and render an industry-only point cloud on top in a distinct color family
- HUD layout: industries on the left, country and local business stats on the right
- Added navigation requirement: the lower HUD section includes clickable backward-navigation options through the current exploration path

## Deferred Alternative

- Keep the two-mode architecture as a documented fallback decision.
- Fallback option: explicit `global view` and `country view` modes with a harder separation of responsibilities.
- Reason to keep it: it is safer, easier to reason about, and may be preferable if the continuous-scene approach becomes too fragile or too expensive to maintain.

## Product Model

The entire experience lives in one globe scene. The app does not hard-switch into a separate country screen. Instead, the same scene gradually morphs as the user interacts.

At far zoom, the user sees a global thematic overlay. Clicking a country fades the rest of the world and biases the camera and rendering toward that country. Zooming further reveals the all-industry business field inside the selected country, then clusters, then raw point markers. The user should feel that they are revealing deeper layers of the same world rather than entering a different app state.

The primary interaction path is:

`global thematic overlay` -> `country selected with world faded` -> `activity map emerges inside country` -> `industry overlay appears when selected` -> `clusters` -> `raw points`

## Layer Stack

### Global Thematic Layer

At far zoom, the globe shows the active world view. This may later include animated border glow, GDP, political status, or business-derived industry analytics. Different world views may come from different datasets, but they should all be rendered through the same layer contract so the globe shell stays consistent.

### Country Emphasis Layer

Once a country is selected, that country receives visual emphasis through opacity, glow, and camera focus. The rest of the globe remains visible but de-emphasized. This layer is the bridge between the global thematic reading and country-level analysis.

### Activity Map

As the camera moves closer, the selected country reveals the all-industry business field. This is not a texture heatmap. It is a dense point field whose own coloring creates the heat effect.

- Hue moves toward red as local business concentration increases
- Brightness increases with aggregated business value

This becomes the main analytical surface for the country when no industry is selected.

### Industry Overlay

When the user selects an industry from the left HUD, the all-industry activity map stays visible as the contextual base layer. A separate industry-only point cloud appears on top using a distinct color family so it reads as a focused analytical overlay rather than a replacement.

### Cluster Layer

At closer zoom, the representation transitions from continuous field reading into grouped concentrations of businesses. This applies to the all-industry set when nothing is selected, or to the selected industry when a filter is active.

### Raw Marker Layer

At the closest zoom tier, cluster groups dissolve into raw business markers. This is the most granular state and should still feel like the natural end of the same visual language.

## Transition Rules

Transitions should be continuous crossfades rather than binary toggles.

- Global thematic signals fade down as country focus increases
- Country emphasis ramps up before business-density layers appear
- The activity map emerges before clusters become legible
- Clusters hand off to raw markers only at close zoom
- Industry selection adds a focused overlay without removing the all-industry base context

Zoom is not only a camera property. It is also a semantic render tier used to blend layers and cap rendering cost.

## Data Products

To keep the continuous-scene model maintainable, the renderer should consume visualization-oriented data products rather than raw business rows only.

### Global Thematic Products

Country-level datasets keyed by country identity or country geometry. Each provider should expose:

- country signal values
- styling metadata
- interaction metadata
- loading and empty-state semantics

These providers may be backed by different sources depending on the active world view.

### Country Analysis Products

The selected country should be driven by a stable payload contract:

- activity points
- industry overlay points
- clusters
- raw business markers

For the first pass, the payload shape should be real even if some values are mocked.

### HUD Products

The HUD should be backed by dedicated products rather than inferred ad hoc from render data.

Left HUD:

- sector and industry list
- counts and ranking cues
- current filter state

Right HUD:

- country-wide totals
- top sectors
- small local hover-area summary

Lower HUD navigation:

- clickable backward-navigation options representing the current path through the experience
- examples: active world view, selected country, selected industry, current zoom/detail tier

The hover-area summary should feel gentle and discreet. It should show local business context around the cursor without competing with the country summary above it.

## State Model

The scene state should center on:

- active global view
- selected country
- selected industry
- zoom tier
- hover area

This state should drive render blending, HUD content, and interaction semantics together.

## Error Handling

The scene should degrade gracefully.

- If a global thematic dataset fails, fall back to a neutral country rendering
- If country analysis data is unavailable, keep country selection and zoom functional while showing a clear HUD data-unavailable state
- If an industry has no data in the selected country, keep the all-industry activity map visible and report the missing coverage in the HUD

## Performance Rules

- Never attempt to show raw markers at country scale
- Use zoom tiers as both UX transitions and performance gates
- Throttle or debounce hover-area updates so they remain subtle and cheap
- Stress the prototype with sufficiently large mocked payloads to expose transition bottlenecks early

## Testing Focus

### Transition Correctness

- selecting a country fades the rest of the world correctly
- the activity map appears in the intended zoom band
- industry overlays appear without replacing the all-industry base layer
- clusters dissolve into raw points at the intended close zoom tier

### State Consistency

- changing global view does not corrupt selected-country state
- clearing an industry filter restores the all-industry path cleanly
- hover-area stats update local context without disrupting country totals
- backward-navigation controls return the user to prior exploration states predictably

### Graceful Fallback

- missing datasets render as degraded but functional states
- empty industries remain understandable
- sparse countries still produce a meaningful visual and HUD presentation

## Prototype Scope

The first implementation should prove the interaction shell rather than the final data pipeline.

- Use real country boundaries
- Use mocked business density and value payloads within countries
- Build the global-to-country-to-business transition system
- Build the HUD composition and backward navigation
- Leave room for the next step to replace mocked analysis payloads with real derived business data
